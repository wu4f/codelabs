Firewalls and filtering allow one to restrict traffic to and from a machine to reduce the attack surface of a machine as well as limit the impact that a compromise might have on the rest of the network. On Linux, iptables provides a mechanism for specifying rules that can be applied to traffic. Consider a web server that should allow incoming connections to ports 80 and 443 as well as outgoing connections to a MySQL database server. It should also allow ssh access, but only to machines in the 131.252.220.0 to 131.252.220.255 range.

Directory listings

One of the common techniques for securing web servers is to disable directory listings when a directory does not have an index HTML file within it. This hides any files or directories that reside in the path that is being accessed. Query your LLMs to provide an nginx configuration that disables directory listings.
